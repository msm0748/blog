# 디스크 크기 변경

## 1단계: GCP 콘솔에서 디스크 크기 변경 (필수)

1. **GCP 콘솔** 접속 후 `Compute Engine` > `VM 인스턴스`로 이동합니다.
2. 좌측 메뉴에서 디스크 클릭
3. 사용 중인 **인스턴스 이름**(`koharu-market-prod`)을 클릭합니다.
4. 상세 페이지 하단의 **수정(EDIT)** 버튼을 클릭합니다.
5. 아래로 내려가 **부팅 디스크(Boot disk)** 항목 옆의 연필 모양 아이콘을 누르거나, 디스크 섹션에서 크기를 **10GB → 20GB**로 수정합니다.
6. 가장 아래의 **저장(SAVE)**을 누릅니다.
    - _참고: 이 작업은 인스턴스를 끄지 않고 실행 중인 상태에서도 가능합니다._

---

## 2단계: 리눅스 터미널에서 용량 인식시키기 (명령어)

콘솔에서 20GB로 늘렸어도 리눅스는 여전히 "나 10GB 아니었어?"라고 생각합니다. 아래 명령어들을 복사해서 입력해 주세요.

### 관련 도구 설치하기

터미널에 다음을 입력해 주세요.

```bash
sudo apt-get update
sudo apt-get install -y cloud-guest-utils
```

### **1. 파티션 크기 업데이트**

먼저 디스크의 파티션이 커진 공간을 다 쓰도록 업데이트합니다.

```bash
sudo growpart /dev/sda 1
```

_(주의: `/dev/sda`와 `1` 사이에 띄어쓰기가 있습니다.)_

### **2. 파일 시스템 확장**

이제 실제로 파일 시스템(용량)을 늘립니다. 아까 보신 `df -h` 결과에 따라 `ext4` 형식이면 아래 명령어를 씁니다.

```bash
sudo resize2fs /dev/sda1
```

---

## 3. 결과 확인

모든 작업이 끝났다면 다시 한번 확인해 보세요.

```bash
df -h
```

**예상 결과:**

- **Size**: 19.x G (약 20GB)
- **Used**: 6.7G (그대로)
- **Avail**: 약 12G (여유 공간 대폭 증가!)
- **Use%**: 약 **35%~40%** (73%에서 뚝 떨어짐)
