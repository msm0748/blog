# pnpm install 시 "Ignored build scripts" 경고 해결 방법

쉽게 말해서 **"이 패키지가 내 컴퓨터에서 수상한 짓(스크립트 실행)을 하려고 하는데, 너 이거 허락할래?"** 라고 물어보는 **보안 승인 절차**입니다.

최근 `pnpm` 버전(v10 이상)에서는 보안 강화를 위해, 패키지를 설치할 때 그 안에 포함된 **빌드 스크립트(내 시스템에 파일을 만들거나 설정을 건드리는 코드)** 가 자동으로 실행되는 것을 막아버립니다.

---

## 1. 이 명령어가 하는 일 (정확한 역할)

패키지 중에는 설치 직후에 추가 작업이 필요한 녀석들이 있습니다.

- **Prisma:** 내 운영체제에 맞는 DB 엔진 파일을 다운로드함.
- **Sentry:** 성능 최적화를 위해 내부 모듈을 컴파일함.
- **NestJS:** 코어 모듈을 빌드함.

`pnpm approve-builds`는 이런 **'설치 후 작업(Post-install scripts)'** 이 필요한 패키지들의 목록을 보여주고, 개발자가 **"응, 이거 안전한 애들이니까 실행해도 돼"** 라고 최종 승인을 해주는 과정입니다.

---

## 2. 승인하면 어떤 변화가 생기나?

이 명령어를 치고 `y`를 눌러 승인하면, `package.json` 파일에 아래와 같은 설정이 자동으로 추가됩니다.

```json
"pnpm": {
  "onlyBuiltDependencies": [
    "@prisma/engines",
    "@sentry/nestjs",
    "@nestjs/core"
  ]
}
```

**이게 기록되면:**

1. 앞으로 `pnpm install`을 할 때마다 다시 물어보지 않습니다.
2. 이 파일을 Git에 올려서 서버에서 받으면, 서버에서도 이 목록을 보고 자동으로 빌드를 진행합니다.

---

## 3. 안 하면 어떻게 되나?

승인을 안 하고 넘어가면 설치는 된 것처럼 보이지만, 정작 앱을 실행할 때 에러가 터집니다.

- **Prisma 에러:** "Query Engine을 찾을 수 없습니다."
- **Sentry 에러:** "Native 모듈을 로드할 수 없습니다."

---

### **요약: 딱 3단계로 기억하세요**

1. **발생:** `pnpm install` 했는데 노란 박스로 "Ignored build scripts" 경고가 뜸.
2. **조치:** `pnpm approve-builds` 입력 후 화살표와 `Space`로 패키지 선택 후 `Enter`, 마지막에 `y` 입력.
3. **결과:** `package.json`에 승인 목록이 저장되고, 막혔던 빌드 작업이 완료됨.

- *결국 "신뢰하는 라이브러리에게 내 시스템 권한 일부를 주는 것"**이라고 이해하시면 정확합니다! 이제 왜 쳐야 하는지 감이 오시나요?
